30 Days of D3: Part One

Home / Blog

Day Four

Today was a long day. I decided that, to really learn D3 at a decent pace, I would need to make multiple "simple" charts each day, for the next few days, so that I can begin prototyping the fun stuff, like interactions, event listeners, and so forth. At some point, I also need to go back to React and keep learning that / learn how to mix React with D3.js

So, anyway, today I made two different, very simple charts. The first chart, using the famous iris dataset contains, for three species of flower, the sepal and petal length and widths for multiple different plants. I plotted this data as a scatterplot, with well-formatted axes. I also represent each flower species with a unique color and a unique symbol. Creating this scatterplot required that I learn D3 tick formatting methods and how to use D3 symbols.

The second chart is a simple bar chart, which displays the frequency of usage for each letter in the English alphabet. This example is adapted directly from Mike Bostock's example on Observable. All I really had to do was pull the dataset (which is attached as a file at the hyperlinked URL, and then format Bostock's code for vanilla JavaScript.

I am also continuing through my learning of the Wattenberger book, but I don't want to always post those examples because I'm not sure of the code's copyright. Tomorrow, I will continue with more "simple" charts, and will also be migrating my code prototyping to Observable, which I have put off learning for too long.

Day Three

Today, I saw a chart on Reddit that shows, for each Harry Potter character, their on-screen time compared to book mentions. Presented as a scatterplot, with a line plotted at a 45 degree angle, it shows which characters are over or underrepresented in the films.

I decided to (very crudely) recreate the chart, by collecting data from JetPunk for book mentions and from Sporcle for on-screen time of each character.

Obviously, there's a lot of things I haven't yet practiced - annotations, coloring dots based on their position above or below the line, and cleaner legends are all things that I plan to learn very soon. I'd also like to improve this chart so that annotations and values appear on mouseover. With the current chart, there's no way to know which dot goes with which character! Totally lame! But I am getting better every day.

Day Two

I know, I know ... starting with a scatterplot on day one is totally lame. Line charts are probably even less interesting. But here it is, a line chart of Bitcoin's value over the last 365 days.

Data in this case was taken from Yahoo's Finance page on Bitcoin (Historical Data for last 365 days, exported as CSV file).

The shaded region represents times when Bitcoin's value has been greater than 15,000 USD per bitcoin. The color of the line and other elements are arbitrary. In an effort to save space, I won't like to my CodePen every time. But you can view the code for all pens at my profile.

Day One

School work has dismantled all the plans I once had to religiously practice D3.js. Data visualization has been an obsession of mine for the last two years, but writing and, before that, research, have scuttled my plans to dive deeply into the topic. This winter, while sitting at home in my New York apartment, I've decided to try, again, to make a concerted effort to practice D3.js on a daily basis. That practice is being done in (daily) coordination with Python and web scraping learning; the goal is to become really well-versed in Python for data cleaning, web scraping, and then D3.js for visualization.

Every day, for the next 30 days, I will update this website. This is part one; it will contain the first ten days of data visualizations. Each day's post will grow in complexity; annotations will emerge, as will more complex color scales and chart types. All visualizations are made with D3.js, and many of the visuals are adapted directly from Amelia Wattenberger's excellent book on the topic.

Now, for day one: a simple scatter plot. The colors grow darker with each successive day of weather (so that the last day of December is the darkest dot, and the first day of the year, January 1, is the lightest). The simplicity of this graph underlies some real complexity, at least in terms of how the D3.js library works. For one, getting it to appear correctly on mobile devices required that I implement the "viewBox" attribute when setting up the wrapper of my visualization. You can view (and edit) all of my code in the CodePen embed, below.

See the Pen 30 Days of D3: Scatter Plot by Niko McCaarty (@nikomccarty) on CodePen.

Finally, with the guiding principles of Edward Tufte's book, and to continue in the spirit of Charles Joseph Minard, I feel it necessary to very clearly state where my data came from, how I obtained it, and to explain any visual elements which might be confusing to viewers. This dataset was obtained directly from Amelia Wattenberger's course. It is the daily weather from every day in New York City, in 2018, as distributed from the DarkSky.net API. The minimum temperature for each day is plotted on the x-axis, while the maximum temperature for each day is plotted on the y-axis. While one might expect a relatively straight line to emerge (which is indeed what we see), denoting that temperatures remain in a relatively narrow range in any given 24 hour period, there are some interesting outliers in the dataset.

The color of each dot represents a date of year; lighter colors are nearer to January 1, 2018, while darker colors are nearer to December 31, 2018. A more intuitive color scale might be "circular"; that is, start and end with the same light color, with a dark color to represent the hot months (May-August or so), since January 1 and December 31 are very near to each other in reality, but appear very different in the chart. In future days of charts, I will expand upon this first chart and make it more intuitive and engaging to explore.
